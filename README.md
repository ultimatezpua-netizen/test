# Простой Telegram Echo Bot

Простой Telegram-бот с эхо-функционалом на Python. Бот повторяет любые текстовые сообщения и отвечает на команду `/start`.

## Функциональность

- **Команда `/start`** - приветствует пользователя
- **Эхо-функция** - повторяет любые текстовые сообщения обратно пользователю

## Требования

- Python 3.8+
- Telegram Bot Token (получить у [@BotFather](https://t.me/botfather))

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd test
```

### 2. Создание виртуального окружения

```bash
python -m venv .venv
source .venv/bin/activate  # На Windows: .venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения

Скопируйте файл `.env.example` в `.env`:

```bash
cp .env.example .env
```

Отредактируйте файл `.env` и укажите ваш токен бота:

```
BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
```

### 5. Запуск бота

```bash
python main.py
```

После запуска вы увидите сообщение:
```
INFO - Запуск бота...
INFO - Бот запущен и готов к работе!
```

## Получение токена бота

1. Найдите бота [@BotFather](https://t.me/botfather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен и вставьте его в файл `.env`

## Использование

1. Найдите своего бота в Telegram по имени, которое вы указали при создании
2. Отправьте команду `/start` для получения приветствия
3. Отправьте любое текстовое сообщение, и бот повторит его обратно

## Рекомендации по безопасности

⚠️ **ВАЖНО**: Соблюдайте следующие правила безопасности:

1. **Никогда не публикуйте токен бота в публичных репозиториях**
   - Токен находится в файле `.env`, который добавлен в `.gitignore`
   - Не коммитьте файлы с токенами в git

2. **Храните токен в безопасности**
   - Не передавайте токен третьим лицам
   - Регулярно обновляйте токен через @BotFather при необходимости

3. **Используйте переменные окружения**
   - На продакшн-серверах используйте системные переменные окружения
   - Не храните секреты в коде

4. **Регулярно обновляйте зависимости**
   ```bash
   pip install --upgrade -r requirements.txt
   ```

## Возможные проблемы и решения

### Ошибка "BOT_TOKEN не найден"
Убедитесь, что:
- Файл `.env` создан и находится в корневой папке проекта
- В файле `.env` указан корректный токен в формате `BOT_TOKEN=ваш_токен`
- Токен скопирован полностью без лишних пробелов

### Бот не отвечает на сообщения
- Проверьте, что бот запущен (в консоли должно быть сообщение "Бот запущен и готов к работе!")
- Убедитесь, что токен действителен
- Попробуйте перезапустить бота
